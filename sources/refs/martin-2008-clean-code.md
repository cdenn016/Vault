# Clean Code

> [!info] Citation
> Robert C. Martin (2008). *Clean Code*. Prentice Hall (Pearson), 2008. ISBN 978-0-13-235088-4.

## TL;DR

A widely-cited practitioner handbook on writing readable, maintainable code: meaningful names, small single-responsibility functions, expressive tests, and disciplined refactoring of 'code smells'.

## Relevance to this research

Software-engineering practice canon informing code-quality standards for the vfe3 implementation (not VFE research theory).
