# Group Equivariant Convolutional Networks

> [!info] Citation
> Cohen, Taco S. and Welling, Max (2016). "Group Equivariant Convolutional Networks." *Proceedings of the 33rd International Conference on Machine Learning (ICML)*, pp. 2990–2999. arXiv:[1602.07576](https://arxiv.org/abs/1602.07576).

## TL;DR

Cohen and Welling generalize the ordinary convolution of a CNN from the group of translations to larger discrete symmetry groups (translations together with rotations and reflections). The resulting G-convolution is provably equivariant: when the input is transformed by a group element, the feature maps transform predictably by the same element rather than being scrambled. This adds weight sharing without adding parameters, lowers sample complexity, and set state-of-the-art results on rotated MNIST and CIFAR-10. The paper is the foundational statement that a symmetry group, declared up front, structures the architecture and its learned representations.

## Problem & setting

A standard convolutional layer is equivariant only to translations: shift the image, and the feature map shifts the same way. But many tasks contain additional symmetries — rotations, reflections — that a plain CNN must learn redundantly from data, wasting capacity and samples. The authors ask how to build layers that are equivariant to a chosen discrete symmetry group $G$ (for images, the wallpaper groups $p4$, generated by translations and 90° rotations, and $p4m$, which adds reflections), so that the symmetry is guaranteed by construction instead of approximated through augmentation.

## Method

The key object is the G-convolution, which replaces the translation group underlying ordinary convolution with a larger group $G$. Concretely:

A feature map is treated as a function on the group $G$ (or, in the first layer, on the plane). The filter is correlated against the input by sweeping it over all transformations in $G$ — every shifted and rotated/reflected copy of the filter — not just spatial shifts. This produces feature maps indexed by group elements. Because the filter bank is shared across the entire group, the layer attains far stronger weight sharing than a plain CNN while using the same number of free parameters. The construction is equivariant to $G$: transforming the input by $g \in G$ permutes the output feature maps by $g$ in a known way. Stacking such layers preserves equivariance, since the composition of equivariant maps is equivariant; pointwise nonlinearities and group-pooling are chosen to respect the action of $G$.

The implementation reduces G-convolution to ordinary convolution by transforming the filters, so the computational overhead over a standard CNN is negligible.

## Key results

On rotated MNIST, G-CNNs ($p4$/$p4m$) substantially reduce error over translation-only CNNs with the same parameter budget, demonstrating that built-in rotational equivariance buys sample efficiency. On CIFAR-10, swapping ordinary convolutions for G-convolutions improves accuracy at matched parameter count, reaching state-of-the-art at publication. The gains come without extra parameters: the symmetry constraint is "free" capacity, realized purely through enhanced weight sharing.

## Relevance to this research

This paper is the conceptual seed of the gauge-theoretic half of the VFE-transformer program. Its central thesis — declare a symmetry group, and let it constrain how representations transform — is exactly the move the project makes when it adopts the block general-linear group $GL(k)$ (`gauge_group block_glk`) as the symmetry acting on per-block latent frames.

Where Cohen and Welling enforce equivariance to a finite group of image symmetries, the VFE transformer aims for equivariance/covariance to $GL(k)$ frame changes (Lie-algebra "phi" parameterization, BCH retraction). G-CNN gives the precise sense in which a transformation of the input should induce a predictable transformation of the features — the property the gauge machinery must preserve under parallel transport and holonomy. G-CNN treats one global symmetry group acting identically everywhere; the program's gauge ambition is the local generalization — a transformation that can differ per token/block — which Cohen's later gauge-CNN work formalizes, making this paper the indispensable predecessor in that lineage (see [[cohen-2019-gauge-cnn]]). G-convolution also organizes features by how they transform under $G$, prefiguring the project's use of irreducible representations, Clebsch-Gordan coupling, and group equivariance when composing $GL(k)$-covariant quantities.

> [!note] Editorial: G-CNN handles compact/discrete groups acting on a homogeneous space; $GL(k)$ is non-compact, so the project cannot import the construction literally. Its value here is the principle (symmetry constrains architecture) and the equivariance bookkeeping, not the specific discrete-group convolution.
